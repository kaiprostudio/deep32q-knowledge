## 🏗️ 使用原則
1. **去個股化**: 這些模式應視為「空白模板」或「思考框架」。
2. **數據導向**: 每個模式均配有對應的 FinMind API 採集清單。
3. **動態更新**: 當發現新的通用風險因子（如：折舊黑洞、匯率敏感度）時，應回寫至對應模式中。
