# F03: 合約負債與營收能見度

- **本題 ID：** Q-F03
- **題目：** 合約負債與未來收入的變動關係為何？訂單轉化為實質營收的速度是否出現停滯？
- **推薦專家：** 財務分析師 (`finance-financial-analyst`)
- **推薦理由：** 專精於「收入認列準則 (IFRS 15)」與「收入建模 (Revenue Modeling)」。能從「合約負債 (Contract Liabilities)」科目中精準讀取訂單能見度、客戶預付款意願，並能透視「虛假訂單」風險，是預判未來營收動能的權威學家。
- **意圖：** 執行「營收領先指標審核」。透過測算「合約負債」與「現行營收」的比例關係及轉化時間，判定企業未來的業績爆發力與營運動能的「含金量」，識別潛在的退單或訂單去化停滯風險。
- **數據路徑與優先權 (Data Source Waterfall)：**
    1. **【核心數據基石：FinMind】**：
       - 調用 `TaiwanStockBalanceSheet` 提取 `CurrentContractLiabilities` (流動合約負債) 之歷史季度數據。
       - 這是審計「訂單能見度」的最上位物理證據。
    2. **【外部驗證：法說會/重大訊息】**：透過 Web Search 檢索「[公司名] 接單 展望」、「合約負債 變動說明」。
    3. **【穿透校準：Annual Report (年報)】**：查閱 `財務報告附註 -> 收入認列會計政策` 及 `重要合約揭露`。獲取訂單履行之關鍵條件。
- **邏輯：** 執行「訂單轉化效率算則」——對比合約負債季成長率與次季營收現實值；執行「能見度週期外推」——根據數據判定未來 1-2 季營收的確定性。
- **來源：** `FinMind (資產負債表)`, `年度報告 (營收認列政策)`, `法說會會議紀錄 (訂單展望數據)`

## 具體化思考點 (智能體強制執行指令)
- **【合約負債現狀核實】**：智慧體**必須**執行自最新年度 $Y$ 溯回至 $Y-3$ 之四年度財務對比分析，列出最近四個季度的「合約負債」金額及其佔營收之百分比。嚴禁進行無數據支撐的「前景看好」評論。
- **【訂單動能定量測算】**：計算「合約負債變動率」與「營收變動率」的彈性。若合約負債大幅萎縮，必須發出「未來營收動能枯竭」之風險警告。
- **【轉化速度壓力測試】**：測算「合約負債 / 日平均營收」比值（預估能見天數）。若該天數異常縮短，智慧體必須判定為「訂單去化過快而新單銜接不足」之風險。
- **【Ticker 校準】**：使用 yfinance 時，必須遵循 `.TW` (上市) 或 `.TWO` (上櫃) 規範，嚴禁輸入純數字。
- **【描述約束】**：**絕對禁用**「模擬」、「預測」、「滿手訂單」。必須使用「根據數據測算顯示」、「訂單能見度趨勢外推」、「合約負債轉換效率分析」。

## 回答模板與注意事項
#### 【F03 專家答詢】財務分析師 (finance-financial-analyst)
- **【專家直斷】品質要求**：須採「指標變動+動能判定」結構（例：合約負債成長 X%，訂單能見度 Y 季，判定未來營收能見度 [明確/模糊]，營收動能處於 [擴張期/衰退期]）。
- **【診斷評級】評分準則**：
    - **S (優)**：合約負債持續增長且轉化為營收的速度加快，顯示訂單獲取能力極強且下游去化順暢。
    - **A (良)**：合約負債保持穩定，訂單能見度維持在安全水位，業務展望清晰。
    - **B (持平)**：合約負債大幅萎縮或轉化天數異常拉長（超過 2 季無營收轉換），預示未來營收可能出現斷崖式下滑或面臨客戶退單風險。
- **【答詢詳解】解析深度**：依照以下三個維度回答：
    1. **訂單領先指標 (Leading Indicator)**：分析合約負債與歷史營收趨勢的過往軌跡對應。
    2. **含金量分析 (Order Quality)**：評估預收款項是否來自高品質、且無退單疑慮的長期客戶。
    3. **產業對位 (Industry Context)**：對比同業（如建商、半導體設備、雲端服務）的平均合約負債水位。
- **【證據來源】數據溯源**：必須精確列出 FinMind 季度數據與對應之資產負債表科目標示。
- **【專家評析】核心分析與洞察內容**：特別關注「長期合約」與「短期合約」的配比。在景氣下行時，合約負債的萎縮是裁員或縮減開支的最早期訊號。專家應在此處穿透營收轉化效率，並給出關於未來業績確定性的實質診斷。
