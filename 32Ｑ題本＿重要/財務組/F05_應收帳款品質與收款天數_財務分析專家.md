# F05: 應收帳款品質與收款天數

- **本題 ID：** Q-F05
- **題目：** 應收帳款 (AR) 的規模與週轉天數 (DSO) 變動為何？是否存在客戶拖欠或虛增營收風險？
- **推薦專家：** 財務分析師 (`finance-financial-analyst`)
- **推薦理由：** 專精於「營運資金效率 (Working Capital Efficiency)」與「信用風險評估」。具備從帳齡分析中識別「呆帳風險」與「營收含金量」的能力。分析師的職責在於確保利潤不僅是帳面數字，而是能如期回收的現金。
- **意圖：** 執行「資產品質與營收真偽審計」。透過測算應收帳款週轉天數與對比營收成長率，識別是否存在「塞貨」或「放寬信用額度衝業績」的激進行為，評估公司的資金回收安全性。
- **數據路徑與優先權 (Data Source Waterfall)：**
    1. **【核心數據基石：FinMind】**：調用 `TaiwanStockBalanceSheet` 提取 `AccountsReceivable` (應收帳款)；調用 `TaiwanStockFinancialStatements` 提取 `Revenue`。
    2. **【穿透校準：Annual Report (年報)】**：查閱 `財務報告附註 -> 應收帳款帳齡分析` 與 `預期信用損失`。獲取逾期款項之真實比例。
    3. **【外部事實檢索：客戶財務狀況】**：若 AR 集中度高（前三大客戶佔比大），則需檢索關鍵客戶之近期信用新聞。
- **邏輯：** 執行「應收帳款比/營收比背離測試」——若 AR 成長速度顯著高於營收成長，且 DSO 天數拉長 > 20%，必須判定為「獲利品質虛胖」；執行「帳齡攤還壓力測試」。
- **來源：** `FinMind (資產/營收)`, `年度報告 (帳齡附註)`, `MOPS 分部資訊 (客戶集中度)`

## 🛠️ 工作流程（按順序執行，不得跳過）

1. **跑工具**：`python3 財務分析工具.py receivable_check [股號]` → 取得 DSO、AR/營收比
2. **判讀 DSO**：DSO <60 天 = 優；60-90 = 正常；>90 = 偏高；>120 = 警戒
3. **對比**：若 DSO 大幅上升但營收成長，需計算 AR 成長率 / 營收成長率，若 >1.5 倍 = 營收品質稀釋
4. **年報補充**：用 `pdf` 工具提取「應收帳款帳齡表」確認有無逾期款項
5. **撰寫報告**：依下方回答模板結構產出

## 具體化思考點 (智能體強制執行指令)
- **【收款天數實測】**：智慧體**必須**計算出具體的「應收帳款週轉天數 (DSO)」。嚴禁模糊描述「收款尚屬正常」。
- **【背離趨勢診斷】**：比對近三季「應收帳款成長率」與「營收成長率」。若兩者發生死亡交叉（AR 升、營收降），必須發出「放帳衝業績」或「客戶支付能力衰退」之警訊。
- **【壞帳準備審查】**：核對年報中「呆帳準備金」提列是否適足。若 DSO 天數拉長但提列比例反而下降，判定為「調節損益之激進會計手段」。
- **【Ticker 校準】**：使用 yfinance 時，必須遵循 `.TW` (上市) 或 `.TWO` (上櫃) 規範，嚴禁輸入純數字。
- **【描述約束】**：絕對禁用「收款良好」、「客戶穩定」。必須量化為具體的「DSO 天數」與「逾期款項佔比 %」。

## 回答模板與注意事項
#### 【F05 專家答詢】財務分析師 (finance-financial-analyst)
- **【專家直斷】品質要求**：須採「收款效率+風險級別」結構（例：DSO 為 X 天，較去年同期拉長 Y 天，[收款效率] 出現停滯，應注意大型客戶 A 之 [信用風險]，獲利品質轉向 [保守]）。
- **【診斷評級】評分準則**：
    - **S (優)**：DSO 天數極短且持續下降，顯示對下游具備強大議價力與優質的客戶回款速度。
    - **A (良)**：DSO 與營收成長同步，帳齡結構健康，無大額逾期款項。
    - **B (持平)**：DSO 異常拉長且 AR 佔營收比重過高，顯示公司可能為了衝營收而過度放帳，資金鏈存有隱憂。
    - **【答詢詳解】解析深度**：必須嚴格遵守 Fact-First 原則，確保每一段論述都有物理數字實體支撐，嚴禁脫離數據空泛推論。
    - **【專家評析】核心洞察**：必須啟用「華爾街空頭視角（Skepticism）」，強制演練最極端之風險阻力，並給出物理對策。
- **【證據來源】數據溯源**：必須精確列出 FinMind `analysis` 收款天數指標與年報帳齡表頁碼。
