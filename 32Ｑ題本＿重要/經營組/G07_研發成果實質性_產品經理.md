# G07: 研發成果實質性

- **本題 ID：** Q-G07
- **題目：** 最近三年研發成果預期項目是否有如期完成？最近三年重大研發成果為何？具備何種變現潛力？
- **推薦專家：** 產品經理 (`product-manager`)
- **推薦理由：** 專精於「產品生命週期 (PLM)」與「技術商品化」分析。能從商業視角判定研發投入是否成功轉化為市場領先的產品，或是僅淪為無效的實驗室數據，識別「研發預算黑洞」。
- **意圖：** 執行「技術變現效率與研發執行力審計」。判定公司是否能如期將技術儲備轉化為營收動能，識別「研發承諾與實質產出」之階差，預警因「研發低效」導致的競爭力衰竭。
- **數據路徑與優先權 (Data Source Waterfall)：**
    1. **【研發進度對位：Annual Report (年報)】**：必須查閱 `業務概況 -> 最近三年度研發成果`。將當年度的產出與三年前規畫的「研發預期項目」進行精確交叉比對。
    2. **【研發效率定量：FinMind】**：調用 `TaiwanStockFinancialStatements` 提取「研發費用 (R&D Expense)」，計算「研發集約度 (R&D Intesity, 研發費/營收)」。
    3. **【穿透校準：MOPS】**：查閱「重大訊息 -> 取得專利/技術認證」。提取實體產出憑證。
    4. **【外部驗證：智權數據庫 (Goofinfo/Google Patents)】**：檢索專利分佈量與核心發明專利佔比。
- **邏輯：** 執行「研發落地率算則」——分析最近三年新增「產品毛利」與三年前「研發投入」之對位關係。若投入激增但新產品毛利貢獻低，判定為「研發迷航」。
- **來源：** `公司年報 (研發進度明細)`, `FinMind (R&D 費用數據)`, `專利數據庫 (趨勢分析)`, `FDA/歐盟認證/車用規範 公告`

## 具體化思考點 (智能體強制執行指令)
- **【研發落地率實測】**：智慧體**必須**列出最近 3 年宣稱完成的 2 個重大項目。若項目僅為「改版」或「優化」，禁止標註為「重大成果」。
- **【變現潛力鑑定】**：分析研發成果是否成功獲取「標竿客戶 (Tier 1)」的訂單。若研發完成但產品遲遲未出貨，必須標註為「技術變現瓶頸」。
- **【無形資產溢價監控】**：核實研發支出是否具備「專利牆」保護效應。若專利分散且無核心主項，判定其技術防禦力為「零散且易被跨越」。
- **【描述指標】**：絕對禁用「研發領先」、「投入巨大」。必須量化為具體的「研發落地成功率 %」、「研發產出比 (ROI)」與「核心專利件數趨勢」。

## 回答模板與注意事項
#### 【G07 專家答詢】產品經理 (product-manager)
- **【專家直斷】品質要求**：點出其研發是「高效落地型」、「防衛性維持型」還是「低效研發型」。判定其技術是否具備 [破壞性創新/漸進式改良] 特徵。
- **【診斷評級】評分準則**：
    - **S (優)**：研發項目準時落地且帶動顯著新營收，技術領先同業一個世代以上，具備強大的技術專利牆保護。
    - **A (良)**：研發產出穩定，能有效支撐現有產品競爭力，認證通過進度符合預期。
    - **B (持平)**：研發項目多次延期，成果缺乏市場競爭力，或研發投入與營收成長完全脫節，存在「資源浪費」之虞。
    - **【答詢詳解】解析深度**：必須嚴格遵守 Fact-First 原則，確保每一段論述都有物理數字實體支撐，嚴禁脫離數據空泛推論。
    - **【專家評析】核心洞察**：必須採「多空雙向」結構——同時提供空方（利空風險）與多方（機會/利多）觀點，每方至少 2 點具體事實，最後綜合權衡。
- **【證據來源】數據溯源**：必須精確標註年報研發進度對比、專利獲准案號、重大認證通過日期。
- **【專家建議】注意事項**：提示投資人關注「研發費用資本化比例」是否異常升高，這往往是用來美化損益表的財務手段。
