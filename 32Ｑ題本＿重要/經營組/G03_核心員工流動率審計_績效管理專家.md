# G03: 核心員工流動率審計

- **本題 ID：** Q-G03
- **題目：** 核心技術或研發團隊的流動率為何？公司如何留住優秀人才？是否有關鍵人才流向競爭對手的跡象？
- **推薦專家：** 績效管理專家 (`hr-performance-reviewer`)
- **推薦理由：** 專精於「組織能力審計」與「人力資本價值評估」。具備從招聘資料、薪酬水位與離職趨勢中讀取企業「技術護城河」是否正在潰散的能力。其核心職責是在業績爆雷前，先從「人才流變」中識別出企業衰敗的早期徵兆。
- **意圖：** 執行「組織韌性與人才護城河審計」。透過量化員工流動率與分析關鍵人才去向，判定企業的技術延續性與知識管理能力，識別因「人才失血」而造成的研發延宕或客戶流失風險。
- **數據路徑與優先權 (Data Source Waterfall)：**
    1. **【第一優先：FinMind】**：調用 `TaiwanStockFinancialStatements` 提取「員工福利費用」並對比員工人數，計算「人均薪資」與「人均獲利」之三年趨勢。
    2. **【第二優先：永續報告書 (CSR/ESG)】**：從 **MOPS** 下載最新的「環境、社會及治理報告」。提取「全體員工離職率」與「核心技術/研發人員離職率」之年度對比數據。
    3. **【第三優先：職場社群與招聘網站】**：檢索 **LinkedIn / 104 人力銀行 / Glassdoor**。觀察關鍵職務的懸掛天數與員工評價。
    4. **【最後手段：Web Search】**：檢索「[公司名] 高管 離職」、「[公司名] 挖角 訴訟」。
- **邏輯：** 執行「HCROI (人力資本回報率) 相關性審計」——計算 `(淨利 + 員工福利費用) / 員工福利費用`。若流動率高但 HCROI 上升，判定為組織排毒；執行「技術空洞化壓力測試」。
- **來源：** `FinMind (人均薪資)`, `CSR 報告 (永續報告書)`, `104/LinkedIn (職位掛缺週期)`, `Glassdoor (組織文化評價)`

## 具體化思考點 (智能體強制執行指令)
- **【人力資本效能檢測】**：智慧體必須計算「人均營收 (Revenue per Employee)」與「人均獲利」。若員工流動率上升但人均產值成長 > 10%，應改寫評語為「組織效率優化」。
- **【核心缺口追蹤】**：智慧體必須查驗 LinkedIn 或 104 是否有大量「關鍵資深職位」同時釋出。若公司核心 R&D 同步離職且被頭號對手錄取，必須發出「技術洩漏與競爭力下降」的嚴厲警告。
- **【薪酬競爭力勾稽】**：對照 FinMind 中的平均薪資與同業基準。若薪資低於同業中位數且流動率高，判定為「人才庫正被競爭對手收割」。
- **【描述指標】**：絕對禁用「員工流動率正常」。必須明確指出「是否足以滿足未來產能或 R&D 擴張需求」。

## 回答模板與注意事項
#### 【G03 專家答詢】績效管理專家 (hr-performance-reviewer)
- **【專家直斷】品質要求**：須採「動態定性+業績關聯」結構（例：該公司員工流動率穩定，且人均獲利成長優於同業，顯示 [組織韌性強] ；雖有局部高管異動，但 [梯隊接班健全] ）。
- **【診斷評級】評分準則**：
    - **S (優)**：流動率極低，人均產值創歷史新高，人才磁吸力強。
    - **A (良)**：流動率處於行業正常範圍，核心技術團隊穩定，招聘達成率高。
    - **B (持平)**：流動率偏高或核心人才頻繁異動，人均產值停滯，面臨知識管理斷代風險。
    - **【答詢詳解】解析深度**：必須嚴格遵守 Fact-First 原則，確保每一段論述都有物理數字實體支撐，嚴禁脫離數據空泛推論。
    - **【專家評析】核心洞察**：必須採「多空雙向」結構——同時提供空方（利空風險）與多方（機會/利多）觀點，每方至少 2 點具體事實，最後綜合權衡。
- **【證據來源】數據溯源**：必須精確列出 CSR 報告頁碼、LinkedIn 觀測到的異動趨勢或 FinMind 薪資對比。
