## [2026-06-04] 班內生管理系統 — 最終設計確認 & 開工

### 最終工作流程（vFinal，經使用者確認開工）

**系統名稱**：班內生管理系統
**架構**：
- 資料庫：`winchamp_enrollments`（MariaDB on NAS 100.97.10.122）
- 表單：公開頁面掛在 port 5999
- 後端管理：掛在 CRM 內聊天室發送填表連結
- 莘莘自動化：Python requests，每次執行先 login 拿 JWT

**系統學號規則**：`[校區前綴] + [入學學年度三碼] + [序號三碼]`
- TY=桃園旗艦校、WL=武陵悅思專校、NL=內壢旗艦校
- 年級→入學年：國三=民國年+1、高一=民國年、高二=民國年−1、高三=民國年−2
- 序號暫從 001 開始（待使用者提供正確數字）

**表單送出自動化（雙軌）**：
1. 寫入本機 DB（status=pending）
2. 本機計算系統學號
3. 登入莘莘拿 JWT
4. POST 學生資料給莘莘（含自編學號）
5. 成功→DB 記錄莘莘 student UUID、status=pushed
6. 失敗→DB 記錄錯誤、status=failed

**後端編輯同步**：後端管理員修改學生資料 → 儲存時自動 PUT 莘莘 API 同步

**後端管理頁面**：類比名冊管理系統搜尋頁面

**模型對話流程**：
- 校區前綴/年級邏輯經多次對接確認
- 學號由本機產生、一併傳莘莘（非先取莘莘學號）
- 莘莘只有兩件事：建立成功（回傳 UUID）、建立失敗
- 後端編輯要 PUT 同步莘莘
